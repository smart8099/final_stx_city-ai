WebSocket Chat Flow

1Widget fetches /api/tenants/{slug}
→
2WS connect :3001 + auth frame
→
3Validate API key → Redis cache
→
4User sends message
→
5Quota check → Response cache check
→
6Load session history (Redis)
→
7streamAgent() via LangGraph
→
8Tavily search on city domains
→
9Stream tokens → browser
→
10Save history + persist to DB

Redis Key Schema

tenant:apikey:{apiKey}

Cached tenant row · TTL 1 day

dept_list:{tenantId}

Cached departments array · TTL 1 day

session:{sessionId}:memory

BaseMessage[] history · TTL 30 min, window k=6

cache:{tenantId}:{sha256(msg)[0:16]}

Response cache (answer + sources) · configurable TTL

quota:{tenantId}:{YYYY-MM-DD}

Daily request counter · TTL resets at midnight UTC
